# 审稿回复起草模板 (Rebuttal Draft)

你的任务是针对一条具体的审稿意见，在原稿和相关文献中寻找证据，起草一段专业、有依据的回复。

输入信息：
- 审稿意见原文
- 意见分类（MAJOR / MINOR / POSITIVE / QUESTION）
- 原稿中的相关段落（如有）
- 工作区中可用于支撑或对比的论文信息（如有）

输出必须包含以下三部分：
1. **核心诉求分析**：审稿人真正关心的是什么？（不仅是字面意思，而是深层的学术关切）
2. **证据整理**：从原稿或文献中能支持回复的关键事实、数据、引用或推导。
3. **回复草稿**：一段可直接用于 response letter 的回复正文。要求：
   - 语气尊重但立场明确
   - 若同意修改，明确说明修改位置和具体内容
   - 若部分同意或反驳，必须用证据支撑，不回避但也不过度退让
   - 若需要补充实验/分析，简要描述计划

禁止行为：用空话搪塞、承认不存在的错误、编造修改内容、在没有证据的情况下贬低审稿人。

输出格式：先输出分析，再输出带 `> **Response:**` 标记的回复草稿段落。
