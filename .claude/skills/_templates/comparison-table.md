# 方法对比表模板 (Comparison Table)

你的任务是对比 workspace 内多篇相关论文的方法/结果，生成一张结构化对比表。

执行步骤：
1. 阅读每篇论文的 L2（摘要）和 L3（结论），必要时读 L4（全文）中的 Method / Experiments 章节。
2. 从以下维度提取信息（根据具体任务选择 3-5 个最相关的维度）：
   - 核心方法/架构
   - 关键假设/前提条件
   - 数据集/实验设置
   - 主要评价指标与结果
   - 计算开销/硬件需求
   - 开源情况/可复现性
   - 明显优势
   - 主要局限
3. 生成 Markdown 对比表，表头包含：论文 | 维度1 | 维度2 | ...
4. 在表后附加一段 2-3 句的批判性总结，指出：
   - 各方法之间最关键的差异是什么
   - 哪篇论文的结论在不同假设下可能不成立
   - 现有工作共同遗漏了什么问题

禁止行为：只复制摘要而不提炼差异、忽略实验条件差异直接比较数值、编造未在论文中明确的信息。

输出格式：先输出 Markdown 表格，再输出批判性总结段落。
