ScholarAIO 配置路线图

这份页面从使用者视角出发，回答的是“我应该按什么顺序把这个项目配起来”。重点不是源码机制，而是： 先用独立 Conda 环境把项目跑通，再配置 LLM、MinerU、本地嵌入和最小数据闭环，最后再理解 MCP 为何只是外部接入层。

推荐主线：Conda 独立环境 LLM：cc switch 与项目配置分开管理 PDF：本地 MinerU 优先，云 API 兜底 先 CLI，后 MCP

二、你当前最适合的具体配置

1. Python 环境

建议固定成一套独立 Conda 环境，不要复用现有科研环境。

conda create -n scholaraio python=3.11 -y conda activate scholaraio python --version
  • Python 3.11 是当前最稳妥选择。
  • 后续 MinerU、embedding、topics 都放这个环境里调试。

2. 项目安装

你是本地源码开发模式，所以直接在仓库根目录 editable install。

pip install -U pip setuptools wheel pip install -e ".[full]"
  • full 会把 embed、topics、import、pdf 相关都装上。
  • 如果只想先跑最小验证，也可以先 pip install -e .

3. 配置文件职责

项目配置建议拆成“公共配置”和“敏感配置”两层。

cp config.local.example.yaml config.local.yaml
  • config.yaml：路径、backend、model、默认策略。
  • config.local.yaml：API key、contact email、Zotero。

4. LLM API 与 cc switch

最好把两套调用面分开：

  • cc switch：管 Claude Code 主 agent 用哪个模型。
  • ScholarAIO：在 config.local.yaml 里单独放内部 LLM key。
llm: api_key: "你的 ScholarAIO LLM Key" ingest: contact_email: "你的邮箱" mineru_api_key: "你的 MinerU Cloud Key"

5. PDF 转换建议

你的目标是本地优先、云端兜底，这也是最合理的生产路径。

  • 本地 MinerU：平时主用，节省云端依赖。
  • 云 API：本地服务没起时自动兜底。
  • 入门期建议先用 Markdown 验证 ingest，再测试 PDF。

6. GPU 与本地嵌入

你有 NVIDIA GPU，可以直接启用本地 embedding。

  • 默认模型是 Qwen3-Embedding-0.6B。
  • 首次下载较大，之后本地缓存。
  • 建议先验证 embed，再碰 topics。
scholaraio embed scholaraio vsearch "你的自然语言问题"

三、首次完整执行顺序

阶段 A：安装与检查

conda create -n scholaraio python=3.11 -y conda activate scholaraio pip install -U pip setuptools wheel pip install -e ".[full]" cp config.local.example.yaml config.local.yaml scholaraio setup check --lang zh

阶段 B：补齐 key 与 backend

  • 在 config.yaml 确定 llm.backend、model、base_url。
  • 在 config.local.yaml 写入 llm.api_key。
  • 可选写入 ingest.mineru_api_key 作为云 fallback。
  • 填 contact_email，提高 Crossref 体验。

阶段 C：最小闭环

scholaraio setup check --lang zh scholaraio --help scholaraio pipeline ingest scholaraio search "关键词" scholaraio show 论文ID --layer 2

建议第一篇先用 Markdown，而不是 PDF。

阶段 D：再开高级能力

scholaraio embed scholaraio vsearch "自然语言问题" scholaraio topics --build

等 CLI 流畅之后，再去研究 scholaraio-mcp 和外部客户端接入。

四、MCP 应该怎么理解

MCP 的本质

  • 它不是新的业务能力。
  • 它是把已有 Python 能力包装成标准 tools。
  • 外部客户端通过 MCP 协议调用这些 tools。

为什么你现在先不用急着接

  • 如果 CLI 没跑通，MCP 只是换个入口报错。
  • 先本地搞清配置、数据目录、依赖与 API。
  • 等最小闭环稳了，再接 Claude Desktop 或 IDE。
配套文档在 explain.md。推荐你先打开这个 HTML 看整体，再照着 explain.md 一步步执行命令。
