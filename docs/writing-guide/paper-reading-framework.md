# 结构化论文精读框架（Paper Deep-Dive Framework）

本框架用于对单篇学术论文进行系统性、批判性精读，适用于文献综述准备、方法对比、研究启发等场景。框架共 20 个条目，分为五个阶段：

- **P1 元数据提取**（条目 1）
- **P2 研究设计解析**（条目 2–8）
- **P3 结果与结论梳理**（条目 9–12）
- **P4 领域关联与价值评估**（条目 13–16）
- **P5 批判性反思与启发**（条目 17–20）

---

## P1 元数据提取

### 1. 文献信息
包括：作者、发表年份、论文标题、期刊/会议名称、卷号、期号、起止页码、DOI 号。

> ScholarAIO 实践：优先从 L1（`meta.json`）提取，缺失时通过 Crossref/S2 API 补全。

---

## P2 研究设计解析

### 2. 核心科学问题
清晰阐述本研究旨在解决的核心科学问题或技术瓶颈。说明这个问题为什么重要，以及它在整个领域中的位置。

### 3. 核心假设与理论构想
明确列出作者提出的核心研究假设、理论模型或概念框架。这些假设是后续全部证据的基石。

### 4. 研究设计
概述整体研究思路：理论推导、仿真模拟、实验验证、案例研究、问卷调查、混合方法等。

### 5. 数据/样本来源
说明数据获取方式、样本规模、选择标准与样本特征。评估样本代表性及潜在选择偏差。

### 6. 方法与技术
详述所使用的关键技术、算法、模型、实验平台、试剂材料或软件工具。关注创新点与技术边界。

### 7. 分析流程
分步说明实验步骤、仿真流程或理论推导的关键环节。此条目的目的是评估研究的可复现性。

### 8. 数据分析
说明所采用的统计方法、有效性验证手段或性能评价指标。关注指标是否合理、对照组是否充分。

> ScholarAIO 实践：在 P2 阶段应主动读取 L4（全文）中的方法章节和公式，必要时编写 Python 代码验证关键推导或数值结果。

---

## P3 结果与结论梳理

### 9. 核心发现
提炼研究中最重要、最创新的科学发现或观测现象。这是作者试图说服读者的"主论点"。

### 10. 实验结果
总结关键的定量结果（性能指标、统计显著性、效应大小）和定性结论。数字必须与原文一致。

### 11. 辅助结果
简述其他支持性的次要结果。评估这些结果是否为" post-hoc "式的勉强附会。

### 12. 最终结论
归纳作者基于证据得出的最终结论。注意区分"实验直接支持"与"作者推测/推广"两个层次。

---

## P4 领域关联与价值评估

### 13. 领域贡献
评价本研究在理论创新、技术突破、方法改进或应用拓展方面对该领域的具体贡献。避免空泛表述。

### 14. 与用户研究的关联
分析该工作与读者的研究主题或综述方向的核心关联。这是决定该论文是否值得深入引用的关键。

### 15. 综述亮点
指出其值得在综述中重点讨论的亮点：开创性、争议性、方法局限性、对未来研究的启示等。

### 16. 图表索引
列出文中所有 Figure 和 Table 的编号、标题及其所展示的核心内容概述，方便快速定位和引用关键证据。

> ScholarAIO 实践：MinerU 解析保留了 `images/` 目录。分析时应打开关键图片确认内容，并在图注中标注数据来源。对于性能图、架构图、流程图，应特别留意坐标轴、样本量、误差棒和图例。

---

## P5 批判性反思与启发

### 17. 个人评价
对该研究的方法严谨性、逻辑完备性和结论可靠性给出个人评价。结合期刊声誉、引用量、作者背景做综合判断，但不盲从权威。

### 18. 疑问与不足
阅读中产生的疑问，或认为其存在的明显不足之处：样本局限、方法缺陷、过度外推、对照缺失等。

### 19. 启发与未来方向
该研究为你带来的新思路、新想法，或未来可探索的方向。可以尝试将文中的方法迁移到你的研究问题上。

### 20. 代表性参考文献
列出文中引用的 1–2 篇最具代表性的参考文献（格式：作者，年份，标题，期刊，卷期，页码），方便追溯重要学术渊源。

> ScholarAIO 实践：在 P5 阶段应调用引用图谱（`scholaraio refs`、`scholaraio citing`、`scholaraio shared-refs`）查找知识库中的相关论文，进行横向对比和交叉验证。将值得跨会话保留的发现写入 `notes.md`（T2 层）。

---

## 使用建议

1. **按需加载**：如果只是想快速判断一篇论文是否相关，先完成 P1 + P2 + P13/P14；只有在确认高度相关后，才深入 P3-P5。
2. **灰暗诚实**：论文结论是作者的"宣称"，不是真理。框架第 17-18 条强制要求批判性视角，避免变成复读机。
3. **图表先行**：遇到实验结果复杂的论文，建议先完成第 16 条（图表索引），再回过来写第 9-12 条，避免在数字中迷失。
4. **跨论文关联**：单篇精读的价值在比较中最大化。读完一篇后，应立即与工作区中方法相近或结论相反的论文做对比。

## 与 ScholarAIO 技能的对应关系

| 框架阶段 | 推荐调用的 ScholarAIO 能力 |
|---------|---------------------------|
| P1 | `scholaraio show <id> --layer 1` |
| P2 | `scholaraio show <id> --layer 4` + 读取 `images/` + Python 代码验证 |
| P3 | `scholaraio show <id> --layer 3` + 读取 `images/` |
| P4 | `scholaraio refs` / `scholaraio citing` / `scholaraio shared-refs` + `scholaraio ws search` |
| P5 | 跨论文对比 + `scholaraio show "<id>" --append-notes "..."` 写入 T2 笔记 |
